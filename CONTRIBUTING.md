# Contribution Guidelines

Thank you for your interest in contributing to the ARKHAN Community! To ensure a smooth and efficient process, please follow the guidelines below.

## How to Report Bugs
1. **Search Existing Issues**: Before creating a new bug report, please search through the existing issues to see if the bug has already been reported.
2. **Provide Detailed Information**: When reporting a bug, please include the following details:
   - Description of the bug
   - Steps to reproduce the issue
   - Expected behavior vs. actual behavior
   - Any error messages or logs

## How to Request Features
1. **Check Existing Feature Requests**: Please check the existing feature requests to avoid duplicates.
2. **Describe the Feature**: Provide a clear and concise description of the feature and its intended benefits or use cases.

## Code of Conduct
We expect all contributors to adhere to our [Code of Conduct](CODE_OF_CONDUCT.md) to ensure a safe and welcoming environment for everyone. Please read it before contributing.

## Support Contact
If you need assistance or have any questions, feel free to reach out to us at our support email:andretijorge2@gmail.com

Thank you for considering contributing to the ARKHAN Community!

# Directrices de Contribución

¡Gracias por tu interés en contribuir a la Comunidad ARKHAN! Para garantizar un proceso fluido y eficiente, por favor sigue las directrices que se indican a continuación.

## Cómo Reportar Errores
1. **Busca en los Issues Existentes**: Antes de crear un nuevo informe de error, por favor revisa los issues ya existentes para ver si el error ya ha sido reportado.
2. **Proporciona Información Detallada**: Al reportar un error, por favor incluye los siguientes detalles:
   - Descripción del error
   - Pasos para reproducir el problema
   - Comportamiento esperado vs. comportamiento real
   - Cualquier mensaje de error o registro (log)

## Cómo Solicitar Funcionalidades
1. **Revisa las Solicitudes Existentes**: Por favor, revisa las solicitudes de funcionalidades ya existentes para evitar duplicados.
2. **Describe la Funcionalidad**: Proporciona una descripción clara y concisa de la funcionalidad y sus beneficios o casos de uso previstos.

## Código de Conducta
Esperamos que todos los colaboradores se adhieran a nuestro [Código de Conducta](CODE_OF_CONDUCT.md) para garantizar un entorno seguro y acogedor para todos. Por favor, léelo antes de contribuir.
